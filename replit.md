# Magazzino Solidale AIM

Gestionale per un magazzino solidale: tracciamento di prodotti/lotti (FEFO), CRM beneficiari, consegne, trasferimenti, volontari e report.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` â€” run the API server (port 5000)
- `pnpm --filter @workspace/magazzino-solidale run dev` â€” run the frontend (port varies)
- `pnpm run typecheck` â€” full typecheck across all packages
- `pnpm run build` â€” typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` â€” regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` â€” push DB schema changes (dev only)
- Required env: `DATABASE_URL` â€” Postgres connection string; `SESSION_SECRET` â€” express-session signing secret

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + wouter + TanStack Query + shadcn/ui + Recharts
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec in `lib/api-spec/openapi.yaml`)
- Build: esbuild (CJS bundle)

## Architecture decisions

- Contract-first API: OpenAPI spec â†’ Orval codegen â†’ typed hooks + Zod schemas used in both FE and BE.
- Stock (giacenze) is computed on-the-fly from `lotti.quantita_residua` aggregated per product+warehouse â€” no separate stock table.
- Transfers are a 2-phase workflow: "avvia" deducts from origin lot, "conferma" confirms receipt at destination.
- Delivery bills (`bolle`) get auto-incremented numeric codes in format `BOLLA-YYYY-NNNN`.
- Beneficiary codice auto-generated from timestamp if not provided by caller.

## Product

- **Magazzino**: CRUD for warehouses, products (multi-category), lots with expiry (FEFO), movements (load/unload), real-time stock levels, inter-warehouse transfers
- **Sociale**: Beneficiary CRM with family dossier, social intervention log, delivery planning calendar, delivery documents (bolle)
- **Logistica**: Volunteer management, vehicle fleet, supplier/donor registry, procurement planning
- **Report**: Stock by warehouse, deliveries by month, deliveries by centro â€” with Recharts charts; every list/entity exportable to XLSX and PDF (client-side: `xlsx` + `jspdf`/`jspdf-autotable`). The Analisi page has a global filter bar: **Periodo** (da/a date range), **Magazzino** (tutti/one), **Centro di Ascolto** (tutti/one). Period drives time-based reports (consegne-per-mese, consegne-per-centro); magazzino filters consegne-per-mese + giacenze; centro filters consegne-per-mese (and client-side narrows consegne-per-centro). FSE+ keeps its own statutory year selector. Report endpoints accept query params `da`/`a` (ISO yyyy-mm-dd; fall back to `anno` then `${currentYear}-01-01..12-31`), `magazzinoId`, `centroAscoltoId` (all optional). Consegne reports join `beneficiari` for the centro filter (consegne have no direct centro column).
- **Report Consegne per Centro**: placed BEFORE the FSE+ report. `GET /report/consegne-per-centro?da&a` groups effettuate consegne by centro di ascolto (via beneficiario), split into **dirette dal centro** (`volontario_id IS NULL`) vs **con volontari** (`volontario_id IS NOT NULL`) + totale. Only `stato='effettuata'` consegne in the date range count ("fatte"). Beneficiari with no centro group under "Senza centro di ascolto" (`centroId=null`). Stacked bar chart + table, XLSX/PDF export.
- **Report FSE+**: end-of-year EU social-fund rendicontazione (`GET /report/fse-plus?anno`). Lists all FSE+ products delivered (qty + total kg) and persons reached. Persons counts ALL household members (titolare + every nucleo member), split by sesso (M/F) Ă— area provenienza (UE/Extra-UE) and adulti/minori. Delivered = bolla `stato IN (confermata,consegnata)` + `lotti.fse_plus=true`, filtered by bolla year. `beneficiariTotali`=distinct families; `personeTotali`=all persons. Null birthdate counts in M/F & UE totals but in neither adulti nor minori. Requires registry fields below.
- **Anagrafica beneficiari**: each person has `sesso` (M/F); `beneficiari.areaProvenienza` (UE/Extra-UE) is FAMILY-LEVEL (nucleo members inherit it). `nucleo_familiare.sesso` per member; etĂ  derived from `dataNascita`. Nucleo members are add/delete-editable in the beneficiary detail page (`DELETE /beneficiari/{id}/nucleo/{membroId}` scoped by both ids); the full anagrafica edit (incl. sesso/area, prioritĂ , consegna a domicilio + note assistenziali) is in EditBeneficiarioSheet (exported from `beneficiario-dettaglio.tsx`). EditBeneficiarioSheet is reused from the Beneficiari LIST page via a "Modifica anagrafica" row action: `QuickEditBeneficiario` fetches the full record with `useGetBeneficiario(id)` before mounting the sheet (avoids overwriting fields not present in the list projection) and invalidates both list+detail queries on save. So prioritĂ  and note assistenziali are editable from both the list and the detail page.
- **Bolle filtri**: the Bolle list is filterable by Magazzino and Centro di Ascolto (combinable â€” intersection). Centro filter joins `beneficiari.centroAscoltoId` (bolle have no direct centro). When a centro filter is active, internal docs (trasferimenti/scarichi) are hidden since they have no beneficiary/centro; the magazzino filter still applies to them client-side.
- **Bolle PDF**: delivery bills downloadable as PDF in 3 selectable templates (standard/moderno/minimal); header shows the beneficiary's Centro di Ascolto name/logo/address, footer shows custom text + association logo. Template + footer set in **Impostazioni Stampa** (singleton settings). The DESTINATARIO/CONSEGNA section shows the recipient name+address (`indirizzoConsegna`||`beneficiarioIndirizzo`)+Cell, the warehouse name+address, and always a "Trasportatore:" line.
- **Bolle trasportatore**: each bolla records who transports it â€” EITHER a volunteer (`volontarioConsegnaId`) OR a free-text `trasportatoreNome` (for self-pickup at the warehouse, default "Ritiro presso il magazzino"). Mutually exclusive; set via a Trasportatore select (active volontari + "Altro") in the create dialog AND the "Chi effettua la consegna" selector in the bolla detail. `BollaDettaglio` joins magazzini (indirizzo/comune) + beneficiari (domicilioâ†’residenzaâ†’comune fallback for `beneficiarioIndirizzo`, telefono) so the PDF can render full addresses.
- **Provenienza lotti**: every lot has a provenance that is EITHER FSE+ (Fondo Sociale Europeo Plus, `lotti.fse_plus=true`, `fornitoreId` null) OR a Fornitore (`fornitoreId` set, `fse_plus=false`) â€” mutually exclusive, enforced server-side in `POST /lotti` (400 otherwise). Both `prodotti` and `lotti` carry an `fsePlus` flag; the product's flag/default supplier pre-selects provenance in the lot-creation forms. Lots are created in TWO places (Lotti "Nuovo Lotto" dialog AND prodotti "Carica in magazzino" CaricoForm) â€” both have the provenance selector; keep them in sync. The Lotti list has a Provenienza column; the prodotti edit sheet shows the product's lots with each lot's provenance (`ProdottoLotti` via `useListLotti({prodottoId})`). Lotti GET joins `fornitori` to return `fornitoreNome`.
- **Lotti**: the Lotti page (read-only tracking list) also has a "Nuovo Lotto" dialog to load a new lot of a product assigned to exactly ONE magazzino. A lot is never splittable across warehouses (enforced by the single `lotti.magazzinoId`); the dialog notes that moving part of the quantity requires a Trasferimento Interno. Create flow mirrors the prodotti "Carica in magazzino" pattern (createLotto + carico movimento; `causale` = donazione/acquisto/rettifica_inventario).
- **Trasferimenti**: create inter-warehouse transfers (origin+destination select, product rows from origin stock capped at available). Each transfer IS its own "bolla di trasferimento" (no separate bolla row â€” `bolle.beneficiarioId` is NOT NULL); on create a "Bolla creata" dialog offers a client-side PDF download. Transfers also appear in the Bolle list as read-only rows tagged "Trasferimento Interno" with a PDF download.
- **Approvvigionamenti (ordini)**: procurement orders with a 3-state workflow `bozza â†’ sottomesso â†’ completato`. New orders save as `bozza` (fully editable via PATCH). "Sottometti" (`POST /approvvigionamenti/{id}/sottometti`) sets `stato=sottomesso` and best-effort emails amministrazione@angeliinmoto.it (helper `artifacts/api-server/src/lib/orderEmail.ts`, wrapped in try/catch so submit never fails). Email sending is currently a logged no-op â€” no sender account configured yet (Gmail integration dismissed by user); wire real delivery in `sendApprovvigionamentoEmail`. After submit a "Completato" button sets `stato=completato`. Orders carry `magazzinoId`+`centroAscoltoId` and are filterable by Magazzino AND Centro di Ascolto (combinable) AND stato. Note field is a large Textarea for the material list and is included in the composed email.
- **Stato filters**: orders, bolle, and consegne lists each have a stato Select filter. Bolle stati: bozza/confermato/consegnato/annullato; consegne stati: pianificata/effettuata; orders stati: bozza/sottomesso/completato.
- **Consegne bolla association/view**: the "Associa bolla" picker (consegne.tsx `bolleDisponibili`) only lists assignable bolle for that beneficiario that are NOT annullato and NOT already `consegnato` (and not linked to another consegna). For an `effettuata` consegna with a linked bolla, clicking the Bolla badge OR the row's "Bolla" action button opens a Sheet rendering the shared `BollaDettaglio` component (exported from `pages/bolle.tsx`) read-only with its built-in "Scarica PDF" download.
- **Scarico magazzino**: unload goods from a warehouse with a document-level `causale` (deteriorata/rubata/scaduta/altro free-text). Auto-decrements stock FEFO over `lotti` and logs `scarico` movimenti for audit. Each scarico IS its own "bolla di scarico" (`SCAR-YYYY-NNNN`) with a client-side PDF download (red accent); appears in the Bolle list as read-only rows tagged "Scarico Magazzino". Create handler is wrapped in `db.transaction` (the one place that uses transactions); `unitaMisura` is derived server-side from the product.
- **Dashboard**: Live KPIs, alerts for expiring lots and low stock, recent movements feed
- **Operatore su bolle**: every bolla, trasferimento, scarico AND intervento records the operator who last touched it via a nullable `operatoreId` FK â†’ `utenti`. Stamped server-side with `req.user!.id` on create AND on every mutating action (bolle: POST/PATCH/righe add+delete/conferma/consegna/annulla; trasferimenti: POST/PATCH/avvia/conferma; scarichi: POST; interventi: POST/PATCH). All list + dettaglio endpoints join `utenti` and return `operatoreId` + a non-personal `operatoreCodice` = `matricola ?? username` (single-row joins use a leftJoin; list endpoints batch-fetch a `utenti` idâ†’codice map). All three PDFs (bolla/trasferimento/scarico) print an "Operatore: <operatoreCodice>" line when present (bolla: under Trasportatore in the right column; trasferimento/scarico: under Note before Firme). The Interventi list shows an "Operatore" column (and it's in the XLSX/PDF export). Interventi auto-synced from a bolla (`syncInterventoBolla`) inherit the bolla's `operatoreId` (the bolla is reloaded fresh after its operator was stamped).
- **Accessi & Profilazione**: session-based internal auth (username/password, bcryptjs hashes). Areas: `generale/magazzino/sociale/logistica/analisi/amministrazione`. Roles (`ruoli`) define allowed `aree` (jsonb string[]) + an `isAdmin` flag (admin = all areas incl. amministrazione). Users (`utenti`) link to a role and carry an optional `matricola` (operator code shown on bolle; falls back to username when empty), editable in the Utenti page. Seed admin (`admin`/`flocap!`, `mustChangePassword=true`) created idempotently at startup (`lib/seedAdmin.ts`). FE auth gate (`src/App.tsx`): no user â†’ Login; `mustChangePassword` â†’ forced ChangePassword screen; else app. Nav (`layout.tsx`) hides non-allowed areas via `hasArea`; backend ALSO enforces via `areaGuard` (route-segment â†’ area map) so hiding nav is not the security boundary. Admin-only "Utenti & Accessi" + "Ruoli" CRUD (`/utenti`, `/ruoli`, both `requireAdmin`). Last-admin lockout is blocked on BOTH user mutations (deactivate/demote/delete last active admin user) AND role demotion (`PATCH /ruoli/{id}` setting `isAdmin=false` on the only admin role with active users). Self-delete blocked. **Idle auto-logout (15 min)**: two layers â€” (1) server session is `rolling: true` with `cookie.maxAge = 15 min` (app.ts), so each request resets the countdown and pure inactivity lets the session lapse; (2) a client idle timer (`src/lib/use-idle-logout.ts`, mounted in `App.tsx` `AuthGate` while `user` is present) listens to real interaction events (mousemove/keydown/scroll/click/touch/wheel + visibilitychange), logs out after 15 min of no interaction, and shows a `common.sessionExpired`/`sessionExpiredDesc` toast. To avoid cutting off active work that makes no network calls (e.g. long form typing), the hook fires a keepalive (`refresh()` â†’ refetch `/auth/me`) at most once per 5 min while the user is active, which resets the rolling server session. Out of scope: email reset, SSO, per-record perms, audit log.

## Gotchas

- Session storage: `connect-pg-simple`'s bundled `table.sql` is NOT picked up by the esbuild CJS bundle, so the session table is defined as a normal Drizzle table (`userSessionsTable`, table name `user_sessions`, cols sid/sess/expire) in `lib/db/src/schema/auth.ts` and the store is configured with `createTableIfMissing:false`.
- Auth cookies are `SameSite=None;Secure` + `trust proxy 1` so the session cookie works inside the cross-site preview iframe. Because of `SameSite=None`, a CSRF Origin/Referer allowlist guard (built from `REPLIT_DOMAINS`/`REPLIT_DEV_DOMAIN`) runs in `app.ts` for all non-GET/HEAD `/api` requests â€” curl POSTs must send a matching `-H "Origin: https://$REPLIT_DEV_DOMAIN"` or they get 403.
- After adding new schema files to `lib/db`, run `pnpm run typecheck:libs` BEFORE checking the API server â€” otherwise stale `.d.ts` declarations miss the new table exports.
- `db.execute(sql\`...\`)` returns a `QueryResult` with a `.rows` property â€” access `result.rows`, not the result directly.
- Decimal columns in Drizzle come back as strings; always `parseFloat()` before sending to the client.
- When passing `enabled` to an Orval-generated hook's `query` options, also pass `queryKey` or TypeScript will error.
- Bolla PDF template names (`standard|moderno|minimal`) live in 4 places (DB column, OpenAPI enum, API `VALID_TEMPLATES`, frontend PDF `ACCENT` map) â€” change them in lockstep + re-run codegen.
